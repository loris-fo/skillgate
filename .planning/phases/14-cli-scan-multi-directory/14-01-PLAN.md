**packages/cli/src/commands/scan.ts** -- Major rewrite:

1. Update ScanOptions interface: add `agent?: string`.

2. Update ScanResult interface: add `agent: DetectedAgent` field.

3. **File discovery logic** (replace the existing defaultDirs approach):
   - Import `AGENT_SCAN_MAP`, `getAgentForPath`, `getAgentDisplayName` from `@skillgate/shared`.
   - Filter AGENT_SCAN_MAP by --agent flag if provided (validate agent name, error if invalid with list of valid values).
   - For each AgentScanEntry, iterate its `paths`:
     - If path ends with `/` (directory): use `glob("**/*.{extensions}", { cwd: resolve(process.cwd(), path) })` to find files. Tag each found file with entry.agent.
     - If path does NOT end with `/` (single file): check if file exists at `resolve(process.cwd(), path)`. If yes, add it tagged with entry.agent.
   - If `--path` is provided: additionally scan that directory with `glob("**/*.md", { cwd })`. Tag results with "unknown" agent (audit engine's detected_agent will label them later).
   - Deduplicate by absolute path (keep the first agent assignment).

4. **No-files behavior** (per user decision):
   - If no files found after discovery, display: "No agent skill files found" + newline + "Directories checked:" + bulleted list of all paths from AGENT_SCAN_MAP + newline + "Run `skillgate install <url>` to add your first skill".
   - Exit code 0 (just return, don't process.exit).
   - In JSON mode: output `{ results: [], directories_checked: [...allPaths], message: "No agent skill files found" }`.

5. **Audit execution**: Keep existing concurrency pattern (CONCURRENCY_LIMIT=5, runWithConcurrencyLimit, rate limit warning at 25+). Include agent in each ScanResult.

6. **Output**:
   - JSON mode: flat array with `{ file, agent, result, error }` per item. Agent comes from the scan map tag (or from audit response's detected_agent for --path files if available).
   - Table mode: call new `printGroupedScanTable(results)` from output.ts.

7. **Gating**: Same logic -- if any result is blocked and --fail is true, exit 1.

**packages/cli/src/lib/output.ts** -- Add grouped scan table:

1. Add new function `printGroupedScanTable(results: Array<{ file: string; agent: string; response?: AuditResponse; error?: string }>)`:

   - After all groups, print summary: `\n{passed} passed, {failed} failed across {groupCount} agents`.

2. Keep existing `printScanTable` function (don't remove -- it may be used elsewhere or for backward compat).
