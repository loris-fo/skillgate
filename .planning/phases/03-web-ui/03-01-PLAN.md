<task type="auto">
  <name>Task 2: Create severity helpers and shared report data loader</name>
  <files>apps/web/src/lib/severity.ts, apps/web/src/lib/report.ts</files>
  <action>
1. Create `apps/web/src/lib/severity.ts`:
   - Import `Score` type from `@skillgate/audit-engine`
   - Export `SEVERITY_CONFIG` as `Record<Score, { color: string; bg: string; percent: number; label: string }>` mapping each score to its Tailwind class names and progress bar percentage:
     - safe: text-severity-safe, bg-severity-safe, 10%, "Safe"
     - low: text-severity-low, bg-severity-low, 30%, "Low"
     - moderate: text-severity-moderate, bg-severity-moderate, 55%, "Moderate"
     - high: text-severity-high, bg-severity-high, 80%, "High"
     - critical: text-severity-critical, bg-severity-critical, 100%, "Critical"
   - Export `VERDICT_CONFIG` as `Record<string, { color: string; bg: string; label: string }>` for each Verdict value:
     - install: severity-safe colors, "Install"
     - install_with_caution: severity-low colors, "Install with Caution"
     - review_first: severity-moderate colors, "Review First"
     - avoid: severity-high colors, "Avoid"
   Follow RESEARCH.md code example exactly.

2. Create `apps/web/src/lib/report.ts`:
   - Import `redis` from `@/lib/kv`
   - Import `AuditResult` from `@skillgate/audit-engine`
   - Import `AuditMeta`, `AuditResponse` from `@/lib/types`
   - Export async function `getReportBySlug(id: string): Promise<AuditResponse | null>`
   - Logic mirrors the GET /api/report/[id] route handler:
     1. Try `redis.get<{ contentHash: string; createdAt: string }>("slug:" + id)` for slug lookup
     2. If not found, try `redis.get<string>("hash-to-slug:" + id)` for hash lookup, then load slug entry
     3. Load audit result from `redis.get<AuditResult>("audit:" + contentHash)`
     4. Build AuditMeta with slug, url, badge_url, created_at, cached: true
     5. Return `{ result, meta }` or `null` if not found
   This avoids the self-fetch anti-pattern (RESEARCH.md Pitfall 2) -- Server Components call this directly instead of fetching their own API.
  </action>
