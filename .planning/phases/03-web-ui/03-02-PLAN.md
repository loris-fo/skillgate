  <name>Task 1: Extend /api/audit route to accept URL-based audits</name>
  <files>apps/web/src/app/api/audit/route.ts</files>
  <action>
Modify the existing POST handler in `apps/web/src/app/api/audit/route.ts` to accept either `{ content: string }` or `{ url: string }`:

1. After parsing the JSON body, check for a `url` field in addition to `content`.
2. If `url` is provided (and content is not):
   - Validate it's a valid URL using `new URL()` in a try/catch
   - Fetch the URL server-side using `fetch(url)` with a 10-second timeout via `AbortSignal.timeout(10000)`
   - If fetch fails or returns non-200, return `errorResponse("VALIDATION_ERROR", "Failed to fetch URL: ...")`
   - If response content-type is HTML (GitHub non-raw URL), return `errorResponse("VALIDATION_ERROR", "URL must point to raw file content, not an HTML page. For GitHub, use the 'Raw' button URL.")`
   - Read the response text as the content
   - Continue with existing flow (size check, dedup, audit, persist)
3. If neither `content` nor `url` is provided, return `errorResponse("VALIDATION_ERROR", "Request body must include 'content' or 'url'")`
4. If both are provided, prefer `content` (ignore url)
5. Keep all existing logic (rate limiting, dedup, slug generation, KV persistence) unchanged

This solves CORS issues (RESEARCH.md Pitfall 4) by fetching URLs server-side.

1. Create `apps/web/src/components/audit-form.tsx` as a Client Component ("use client"):
   - State: `content` (string), `url` (string), `loading` (boolean), `error` (string | null)
   - Import `useRouter` from `next/navigation`, `AuditResponse` and `ErrorResponse` from `@/lib/types`

   **Layout (per locked decisions):**
   - Stacked layout: URL input field on top (compact), large textarea below
   - Both inputs always visible -- user picks whichever method they prefer
   - URL input: single-line `<input type="url">` with placeholder "https://github.com/user/repo/raw/.../SKILL.md"
   - Textarea: large multi-line textarea (min 12 rows) with placeholder "Paste SKILL.md content here..."
   - Character count displayed below textarea (text-text-muted, right-aligned)
   - Submit button: purple accent (bg-accent hover:bg-accent-hover), full width or right-aligned, disabled when both inputs empty or loading

   **Validation (client-side, per locked decisions):**
   - If URL is filled, validate format with `new URL()` before submit. Show inline error "Invalid URL format" if invalid.
   - If textarea is filled, just check non-empty
   - If both are filled, send `{ content }` (content takes priority)
   - If neither is filled, disable submit button

   **Submit flow:**
   - POST to `/api/audit` with `{ content }` or `{ url }` based on which input is filled
   - On submit: set loading=true, error=null
   - On success: `router.push(\`/report/${data.meta.slug}\`)`
   - On error: parse ErrorResponse, set error message, set loading=false

   **Loading state (per locked decisions):**
   - Full-page loading takeover: form fades out (opacity transition), centered loading state takes over
   - Loading state: show a pulsing/spinning indicator with "Auditing skill..." text
   - Use CSS transitions (animate-fade-in from theme), no flashy animations

   **Error state:**
   - Show error message above the submit button in a rounded container with red text
   - Clear error when user starts typing again

   **Styling:**
   - Use dark theme classes: bg-surface-1 for form container, bg-surface-2 for inputs, border-border
   - text-text-primary for labels, text-text-secondary for placeholders
   - font-mono for the textarea (SKILL.md content is code)
   - Rounded corners, padding, subtle transitions
