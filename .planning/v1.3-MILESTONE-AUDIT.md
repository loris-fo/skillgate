### Integration Gap (1 issue)

**CLI Zod schema missing `detected_agent`** (`packages/cli/src/lib/schema.ts`)

The CLI's local `auditResultSchema` does not include the `detected_agent` field added by Phase 13. Zod strips unknown keys by default, so `detected_agent` in API responses is silently dropped before reaching CLI output functions.

**Impact:**
- `skillgate install <url>` — agent suffix `(Cursor rules file)` never appears in CLI output
- Install command's local URL fallback detection runs unconditionally (benign but semantically wrong)
- `skillgate scan` agent grouping is **unaffected** (uses path-derived agent from AGENT_SCAN_MAP)

**Fix:** Add `detected_agent: z.enum(["claude", "cursor", "windsurf", "copilot", "cline", "aider", "unknown"]).optional()` to `auditResultSchema` in `packages/cli/src/lib/schema.ts`.

**Affected requirement:** DETECT-02 (CLI display path only; web path works correctly)
